## Maßnahmenplan in Wellen

- **W1 — Vertrags-Neufassung (eine breaking Welle, vor 1.0-Freeze):** 0006 + 0007 +
  0008 + 0010 + 0011 + 0013 + 0021 zusammen: FindingStatus, CheckOutcome (abgeleitet),
  Erfüllungsgrad statt Prozent, `finding_key` (HMAC, `NIS2SCAN_SECRET`),
  Pseudonymisierung raus aus den Checks (eine zentrale Implementierung im
  Report-Export statt ~30 Duplikate), Evidence-Allow-List formalisieren,
  `schema_version`/`mapping_version`/`rechtsstand`, tote `severity_threshold`
  streichen. API-Schemas und Frontend ziehen nach. Singleton-Frage (Registry) klären.
- **W2 — Repo-Split (0014/0019):** pdf/remediation/monitoring/licensing →
  `nis2scan-premium`; Zuordnung api/+frontend entscheiden (s. u.);
  Entry-Point-Loader (`nis2scan.plugins`) im freien CLI.
- **W3 — Fail-Safe & Report (0016/0009/0012/0004):** konservative Outcome-Ableitung,
  Prüfgrenzen pro Check, Attestierungs-Checkliste, Abdeckungstexte, Disclaimer +
  „Selbsteinstufung", Kategorie-Vokabular `wichtig|besonders_wichtig`.
- **W4 — Rechts-Review-Kampagne (0018):** Mapping mit Quellennachweisen nachrüsten
  (gesetze-im-internet.de, EUR-Lex, BGBl), dann alle Checks batch-weise pro §30-Nr.
  über alle Provider durch das Vier-Augen-Review (Gründer + `legal-reviewer`-Agent).
- **W5 — Konsolidierung:** CONTEXT.md, ADRs, Skills, Agent aus dem Planning-Repo
  hierher migrieren; beide CLAUDE.md mergen; ADR-0017 revidieren; DoD-0.1-Gate
  (ADR-0020) abhaken; GTM-Timeline neu schneiden.

## Offene Gründer-Entscheidungen

1. **GCP:** 51 fertige Checks, aber in keinem ADR und ohne Rechts-Review.
   Empfehlung: Code behalten; 0.1-Kommunikation nennt AWS + Azure; GCP läuft als
   „experimental" mit sichtbarem Banner im Report, bis W4 auch GCP durchlaufen hat.
   ADR-0017 entsprechend revidieren.
2. **api/ + frontend/ (SaaS):** Laut Tier-Tabelle Enterprise — gehören nicht ins
   öffentliche Apache-Repo. Empfehlung: eigenes privates Repo `nis2scan-saas`
   (eigene Deploy-Einheit, anderes Release-Tempo als das CLI-Plugin).
3. **Feature-Matrix:** Skill-Version vs. Repo-Version divergieren — welche gilt?
